# Your first credential

This walkthrough gets you to your first issued credential in minutes.

Target credential for this tutorial:

- `skillcraft-gg/hello-world`

## What this tutorial covers

You will:

- Install and initialize Skillcraft in a repo.
- Generate verifiable evidence with a normal commit.
- Check that evidence is ready to claim.
- Claim `skillcraft-gg/hello-world` and verify claim status.

## Prerequisites

- Node.js + npm available on your machine.
- A local git repository with at least one commit.
- Internet access for verification workflows.

## Step 1: Install and initialize

```bash
npm install -g skillcraft
cd your-project
skillcraft enable
```

## Step 2: Make a tracked change

Any normal code change is fine. Commit it to capture your first evidence item.

```bash
touch .skillcraft-guide.md
git add .skillcraft-guide.md
git commit -m "chore: add first skillcraft evidence"
```

If this repository was just initialized, any first commit is valid for the tutorial.

## Step 3: Confirm progress

```bash
skillcraft progress
```

You should see progress entries tied to your local commits.

## Step 4: Claim `skillcraft-gg/hello-world`

```bash
skillcraft claim skillcraft-gg/hello-world
```

`hello-world` is intentionally simple and only requires at least one qualifying commit (`min_commits: 1`).

## Step 5: Validate the result

You should see one of these states:

- `pending`
- `processing`
- `verified`
- `issued`
- `rejected`

If the credential is still processing, wait a few moments and check again.

## Troubleshooting your first attempt

- **No evidence appears in progress:** make sure you are in the same repository where `skillcraft enable` was run and that you committed at least one change.
- **Claim command errors:** verify you used `skillcraft-gg/hello-world` exactly.
- **Long-running verification:** processing usually resolves once proof checks complete.

For deeper troubleshooting patterns, see the [Troubleshooting reference](../reference/troubleshooting).
