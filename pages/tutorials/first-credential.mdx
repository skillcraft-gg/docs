# Your first credential

This walkthrough gets you to your first issued credential in minutes.

Target credential for this tutorial:

- [`skillcraft-gg/hello-world`](https://skillcraft.gg/credentials/skillcraft-gg/hello-world/)

## What this tutorial covers

You will:

- Install Skillcraft and confirm OpenCode or OpenAI Codex is available.
- Enable Skillcraft in a repository and start tracking [`skillcraft-gg/hello-world`](https://skillcraft.gg/credentials/skillcraft-gg/hello-world/).
- Generate the change through a supported AI coding agent so Skillcraft can capture the required evidence.
- Commit and push that change to your public GitHub repository.
- Check that evidence is ready to claim.
- Claim [`skillcraft-gg/hello-world`](https://skillcraft.gg/credentials/skillcraft-gg/hello-world/) and verify claim status.

## Prerequisites

Complete the [Installation](../installation) prerequisites before starting. In particular, make sure `gh` is authenticated with `gh auth login` and either `opencode` or `codex` works in your shell.

For this tutorial, your GitHub account needs permission to push to the public GitHub repository you are using.

## Step 1: Install and enable Skillcraft

<Tabs items={['OpenCode', 'Codex CLI']}>
  <Tabs.Tab>

```bash
npm install -g skillcraft
cd your-project
skillcraft enable --agent opencode
```

  </Tabs.Tab>
  <Tabs.Tab>

```bash
npm install -g skillcraft
cd your-project
skillcraft enable --agent codex
```

  </Tabs.Tab>
</Tabs>

If both OpenCode and OpenAI Codex are installed, `skillcraft enable` without `--agent` asks which integrations to enable.

## Step 2: Track the credential first

```bash
skillcraft progress track skillcraft-gg/hello-world
```

## Step 3: Run work through your coding agent

Run your enabled agent after `skillcraft enable` so Skillcraft can capture the tutorial evidence from an agent-backed workflow.

<Tabs items={['OpenCode', 'Codex CLI']}>
  <Tabs.Tab>

```bash
opencode run "Create a file named hello.md containing a single line: Skillcraft tutorial evidence."
```

  </Tabs.Tab>
  <Tabs.Tab>

```bash
codex exec "Create a file named hello.md containing a single line: Skillcraft tutorial evidence."
```

  </Tabs.Tab>
</Tabs>

You can alternatively run the agent TUI and provide the commands there.

## Step 4: Commit and push the result

Your coding agent creates the change, but you still need to commit and push it.

```bash
git add hello.md
git commit -m "feat: create first skillcraft tutorial evidence"
git push
```

## Step 5: Confirm progress

```bash
skillcraft progress
```

You should see progress entries tied to the agent-backed commit in this repository.

## Step 6: Claim [`skillcraft-gg/hello-world`](https://skillcraft.gg/credentials/skillcraft-gg/hello-world/)

```bash
skillcraft claim skillcraft-gg/hello-world
```

## Step 7: Validate the result

Track the status of your claim with:

```bash
skillcraft claim status skillcraft-gg/hello-world
```

You should see one of these states:

- `pending`
- `processing`
- `verified`
- `issued`
- `rejected`

If the credential is still processing, wait a few moments and check again.

Once your claim is verified, your profile will be added to the issued list for this credential:

- [Issued credentials for Hello World](https://skillcraft.gg/credentials/skillcraft-gg/hello-world/issued/)

Your credential will also appear on your profile and be issued at its own URL, for example:

- [Example credential profile](https://skillcraft.gg/credentials/profiles/github/blairhudson/)
- [Example issued credential page](https://skillcraft.gg/credentials/profiles/github/blairhudson/skillcraft-gg/hello-world/)

From there you can [verify it](https://skillcraft.gg/credentials/profiles/github/blairhudson/skillcraft-gg/hello-world/?verify).

## Step 8: Share it on LinkedIn

Once your credential has been issued, you can [share it on LinkedIn](https://skillcraft.gg/credentials/profiles/github/blairhudson/skillcraft-gg/hello-world/?share).

## Troubleshooting your first attempt

- **No evidence appears in progress:** make sure you ran `skillcraft enable`, then `skillcraft progress track skillcraft-gg/hello-world`, then your agent command, and committed the resulting change in the same repository.
- **`gh` is not found or not authenticated:** install GitHub CLI and run `gh auth login`, then retry the claim flow.
- **No supported agent is found:** install OpenCode or OpenAI Codex first, then rerun the tutorial.
- **Claim does not verify:** make sure the commit was pushed to the public GitHub repository before claiming.
- **Claim command errors:** verify you used [`skillcraft-gg/hello-world`](https://skillcraft.gg/credentials/skillcraft-gg/hello-world/) exactly.
- **Long-running verification:** processing usually resolves once proof checks complete.

If needed, use the [Getting started guide](../getting-started), [OpenCode reference](../reference/ai-coding-agents/opencode), [Codex CLI reference](../reference/ai-coding-agents/codex-cli), and [CLI reference](../reference/cli).
