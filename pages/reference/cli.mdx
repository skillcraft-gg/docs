# CLI reference

Use this as the canonical list of commands from `skillcraft --help`.

```bash
skillcraft [options] [command]
```

`skillcraft` is the top-level CLI command for repository enablement, evidence capture workflows, skills/loadout publishing, claims, and local verification.

## Global flags

```bash
--json        machine-readable JSON output
-h, --help    display help for command
-v, --version output version number
```

`--json` is supported by commands that include structured output.

## `skillcraft enable`

Initialize Skillcraft in the current repository.

```bash
skillcraft enable
skillcraft enable --agent opencode
skillcraft enable --agent codex
skillcraft enable --agent opencode --agent codex
```

If one supported agent is installed, `skillcraft enable` uses it automatically. If both OpenCode and OpenAI Codex are detected, Skillcraft prompts unless `--agent` is supplied. `--agent` is repeatable and also accepts comma-separated values.

Skillcraft installs its git hooks into the repository's active hooks path, so it works with repo-local setups such as Husky's `.husky` directory. If a repository disables hooks entirely with `core.hooksPath=/dev/null`, `skillcraft enable` fails until hooks are re-enabled.

## `skillcraft disable`

Disable Skillcraft for the current repository and remove local integration state.

```bash
skillcraft disable
skillcraft disable --agent opencode
skillcraft disable --agent codex
```

Use `--agent` to remove one integration while leaving others enabled for the same repository.

## `skillcraft status`

Show repository Skillcraft status: repository detection, enabled state, pending skills, hook presence, proof metadata, and recent evidence counts.

```bash
skillcraft status
```

## `skillcraft doctor`

Run installation and environment readiness checks.

```bash
skillcraft doctor
```

The doctor output reports both `opencode` and `codex`, along with the detected supported agents.

## `skillcraft learn`

Start a guided Learn Mode session in the current git repository.

```bash
skillcraft learn
skillcraft learn --agent opencode
```

If OpenCode is installed, `skillcraft learn` prepares a temporary guided session and launches the supported agent for the repository root. If both OpenCode and Codex are detected, Skillcraft prompts for agent selection, but Codex Learn Mode is not enabled yet.

During the session, Skillcraft writes temporary local guidance files so the agent behaves like a learning coach. When the session exits, Skillcraft removes those temporary files and prints a reminder that Learn Mode is disabled until you run `skillcraft learn` again.

If the repository is not already enabled for Skillcraft evidence capture, Learn Mode tells the agent to mention `skillcraft enable --agent opencode` as an optional next step, but it should not run that command unless you explicitly ask.

## `skillcraft repos`

Manage tracked repositories.

```bash
skillcraft repos list
skillcraft repos prune
```

`repos list` shows tracked repository entries.

`repos prune` removes unavailable repository entries from tracking.

## `skillcraft progress`

Show credential readiness across tracked repositories.

```bash
skillcraft progress
```

```bash
skillcraft progress --refresh
```

`--json` is supported and outputs machine-readable status for tracked credentials.

```bash
skillcraft progress track <credential-id>
skillcraft progress untrack <credential-id>
```

- `skillcraft progress` evaluates all locally tracked credentials and prints eligible/blocked state per credential.
- `skillcraft progress --refresh` refreshes the credential index cache before evaluation.
- `skillcraft progress track <credential-id>` begins tracking a credential for progress checks.
- `skillcraft progress untrack <credential-id>` stops tracking a credential.

The credential index is cached locally as a timestamped snapshot and automatically refreshed about every 6 hours. Use `--refresh` if you want to force an immediate re-download.

## `skillcraft skills`

Manage local skill workflows and registry discovery.

```bash
skillcraft skills add <id>
skillcraft skills publish <owner-slug>
skillcraft skills validate
skillcraft skills list
skillcraft skills inspect <id>
skillcraft skills search [query] [--source <source>] [--limit <n>]
```

- `skills add <id>` installs a local or external skill into `.agents/skills` for the current project.
- `skills publish <owner-slug>` publishes a local skill to the registry.
- `skills validate` checks local skill layout.
- `skills list` prints detected skills.
- `skills inspect <id>` prints registry details for a skill.
- `skills search [query]` browses the published index.
  - `--source <source>` filters by source.
  - `--limit <n>` caps displayed results.

Installed skill evidence is recorded when a supported coding agent actually uses the installed skill.

## `skillcraft verify`

Verify local proof references and proof-object integrity.

```bash
skillcraft verify
```

## `skillcraft claim`

Claim a credential or inspect claim details.

```bash
skillcraft claim <credential>
skillcraft claim --all-repos
skillcraft claim --repo <path...>
skillcraft claim list
skillcraft claim status <issue>
```

- `claim <credential>` submits a claim using the credential identifier.
- `--all-repos` includes all tracked repositories.
- `--repo <path...>` includes explicit repository paths.
- `claim list` shows issues in the credentials repository.
- `claim status <issue>` shows issue status by number.

## `skillcraft loadout`

Manage active loadouts for local evidence context.

```bash
skillcraft loadout use <id>
skillcraft loadout clear
skillcraft loadout share <id>
```

- `loadout use <owner>/<slug>` activates a loadout.
- `loadout clear` clears active loadouts.
- `loadout share <owner>/<slug>` publishes a local loadout.

## `skillcraft help`

Built-in command reference helper.

```bash
skillcraft help [command]
```

Provides usage details for a specific command.
