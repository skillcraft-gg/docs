# GitHub

## Status

Supported.

## What works today

- Local repository enablement and claim workflows in GitHub-hosted repositories.
- Evidence capture tied to local repository state and commit history.
- Existing CLI-based claim status and progress checks.

## Setup

Use the normal CLI onboarding flow for a repository (`skillcraft enable`, commit, claim).

## README badge

After you enable Skillcraft for a repository, you can add the hosted Skillcraft badge to your GitHub README:

[![Skillcraft Enabled](https://skillcraft.gg/badges/enabled.svg)](https://skillcraft.gg/docs/)

```md
[![Skillcraft Enabled](https://skillcraft.gg/badges/enabled.svg)](https://skillcraft.gg/docs/)
```

This links readers to the Skillcraft docs and uses the public badge asset hosted at `https://skillcraft.gg/badges/enabled.svg`.
