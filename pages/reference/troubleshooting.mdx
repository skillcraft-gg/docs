# Troubleshooting

Common issues during onboarding and first-credential setup.

## Command not found

If `skillcraft` is unavailable in your terminal:

- Reopen your shell after install.
- Confirm your PATH includes npm global bin directories.
- Verify with `skillcraft --version`.

## `skillcraft progress` shows no evidence

- Confirm you ran `skillcraft enable` in the same repository.
- Ensure you have at least one committed change after enabling.
- Check that command output is for the current repository path.

## Claim shows errors for `skillcraft-gg/hello-world`

- Confirm the credential id uses the exact canonical form.
- Run `skillcraft progress` first; claim only after evidence is listed.
- Retry after any active network pause or verification queue delay.

## Claims stay `processing` too long

- Most delays are transient during verification.
- Retry claim status after a short wait.
- If failure persists, check remote verifier health and run:

```bash
skillcraft progress
skillcraft claim skillcraft-gg/hello-world
```
